# Screen Selection

Choose three to six screens that explain the app without requiring the viewer to infer the workflow.

## Selection order

1. Lead with the clearest primary benefit.
2. Show the main action a user performs.
3. Show the result, history, analysis, organization, or convenience produced by that action.
4. Add supporting differentiators only when they are visually distinct and verified.

## Prefer

- screens with meaningful, safe sample data
- states that look complete rather than empty
- one clear value proposition per image
- a coherent progression across the set
- stable screens that can be reproduced for Japanese and English outputs

## Avoid

- login, account, legal, permission, or loading screens unless they are the product value
- screens containing personal or production data
- multiple images that communicate the same benefit
- features that are unfinished, unreleased, or unavailable to store users
- claims that depend on information not visible in the screenshot or verified in the app

Record the selected screen, intended message, navigation path, required sample data, and capture status before generation.
