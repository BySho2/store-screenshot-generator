# Copy Guidelines

## Headline

- communicate one user benefit, not a generic feature label
- keep it readable in one or two short lines
- use concrete language supported by the screen
- avoid exaggerated claims such as best, perfect, effortless, or guaranteed unless substantiated

## Supporting sentence

- explain what the user can do or understand on the shown screen
- add information rather than repeating the headline
- keep the sentence short enough to remain legible at store preview size

## Localization

- write Japanese for a Japanese reader and English for an English reader
- preserve meaning and product terminology without forcing identical line breaks
- use the app's established labels when repository or UI evidence exists
- do not translate trademarks, product names, or feature names without evidence

## Set consistency

- use the same voice and level of formality throughout
- avoid presenting multiple slides with the same promise
- order slides so the first image works as the strongest standalone message
