# QA Checklist

## Source screenshots

- intended screen and state
- no personal, customer, credential, or production data
- no debug UI, keyboard, notification, permission dialog, or loading overlay
- consistent device, orientation, locale, appearance, time, and sample data

## Generated images

- correct store dimensions and RGB PNG format
- all expected store, locale, and slide combinations exist
- no clipped headline, body, footer, or screenshot
- screenshot aspect ratio is preserved
- sufficient contrast and readable text size
- consistent spacing, hierarchy, colors, and screenshot scale
- copy accurately describes the displayed screen
- Japanese and English are natural and free of placeholder text

## Delivery

- selected order is intentional
- automated and visual checks are both reported
- remaining caveats are explicit
- no store upload or publication occurred without separate authorization
